## Highlights

The first working release of Project Santara. The Python intelligence tier and the Go performance tier are both built, tested, and wired together. The first anchor problem ("Apa yang terjadi ke inflasi kalau Pertamax naik 30 persen lagi?") is answered end to end through A2A JSON-RPC at sim-gateway, HTTP to sim-id-fiskal, and gRPC to sim-engine.

## Added

- **sim-kernel** - Nine modules: models, events, errors, locales, telemetry, a2a, mcp, prompts, grpc_contracts. 21 tests pass, 88% line coverage. Apache 2.0.
- **sim-engine (Go 1.22+)** - gRPC server implementing all 9 RPCs from simulation.proto. In-memory state with proper RWMutex semantics (locked and unlocked snapshot paths to avoid non-reentrant deadlock). Tick engine (v0.1.0 is a counter, gated on benchmarks). Zerolog telemetry. 4 in-process Go tests pass.
- **sim-gateway** - FastAPI app with GET /healthz, GET /.well-known/agent-card.json, and POST /a2a. JWT HS256 bearer validation. Forwards questions to sim-id-fiskal. 7 tests pass.
- **sim-id-fiskal** - FastAPI app with GET /healthz and POST /ask. Pass-through fiscal model: Pertamax 10%, Pertalite 30%, Solar 20%. Answers the first anchor question. 7 tests pass.
- **rpc-contracts** - simulation.proto with 10 RPCs and 11 messages. Generated Python stubs. End-to-end integration test that starts the Go binary and exercises all 9 RPCs. 5 tests pass.
- **Docker Compose** - sim-engine, sim-gateway, sim-id-fiskal wired together at the repository root.
- **GitHub Actions CI** - validate workflow with four jobs (Lint, Test, End-to-end, Container images). Lint and test run on every push and PR. Container images build and push to GHCR (and Docker Hub if secrets are configured) on tags.
- **Hugging Face dataset** - raihanpka/indonesia-fiscal-pressure with 56,709 rows, 69 indicators, 2014-11-18 to 2026-06-10.

## Known limitations (v0.1.0 pre-alpha)

- The Go tick engine is a counter, not a simulation. The Run RPC increments tick by count. The real tick logic (hunger, supply, demand, prices, agent decisions) is gated on benchmarks proving the Python tier cannot do it fast enough.
- The fiscal pass-through model is a single coefficient per fuel. Subsidi interactions, second-round effects, and exchange rate pass-through are not yet modeled.
- JWT validation is a signature check only. No audience, no issuer, no expiration enforcement, no key rotation.
- No persistent state. In-memory only. PostgreSQL via pgx is planned for v1.5.0.
- The Hugging Face dataset is published but not yet wired into the runtime services.
- The other sim-id services (politik, iklim, agraria) are still scaffolds. Phase 2 will fill them in.

## Installation

```bash
git clone https://github.com/raihanpka/project-santara
cd project-santara
make install
make test
```

## Pull images

```bash

docker pull ghcr.io/raihanpka/sim-gateway:0.1.0
docker pull ghcr.io/raihanpka/sim-id-fiskal:0.1.0
```
